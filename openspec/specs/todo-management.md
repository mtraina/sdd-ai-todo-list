# Todo Management Specification

## Purpose

Allow users to manage personal todo items.

## Requirements

### Create Todo

Users can create a todo item.

Fields:

* title
* description

### View Todos

Users can view all todo items.

### Complete Todo

Users can mark a todo item complete.

### Delete Todo

Users can delete a todo item.

## Acceptance Criteria

* Todos persist in database.
* Completed todos show completed status.
* Deleted todos are removed.
* API returns proper HTTP status codes.
